# LoopGuard 管理页面交互优化设计

日期：2026-05-29
状态：已确认设计，待实现

## 1. 背景

当前管理页面（程序管理、用户管理、API Key 管理）存在多处交互断点，admin 操作不顺畅。核心问题：

- **审批人手填数字 ID**：需要跨页查询用户 ID，流程中断
- **参数白名单手写 JSON**：极易格式错误
- **操作无反馈**：成功/失败静默，用户不确定操作是否生效
- **API Key 无法管理**：只能创建，无法禁用/启用/删除
- **创建用户无密码确认**：输错无法察觉
- **按钮无 loading 态**：可能重复提交

## 2. 改动范围

### 前端

| 页面 | 改动项 |
|------|--------|
| 程序管理 - 表格 | 审批人列显示头像+名字（非裸 ID）；二进制路径降为副信息行；新增超时列；操作列增加"启用/禁用"快捷切换 |
| 程序管理 - 创建弹窗 | 表单分三区（基本信息/审批配置/参数白名单）；审批人改为下拉选择用户；参数白名单改为动态 key-value 表单 |
| 程序管理 - 编辑弹窗 | 启用状态改用 Switch 组件；审批人改为下拉选择；附带说明文字 |
| 用户管理 - 表格 | 头像+用户名（去掉 ID 列）；当前登录用户高亮；其他用户增加"重置密码"操作 |
| 用户管理 - 创建弹窗 | 新增"确认密码"字段 |
| API Key 管理 - 表格 | 新增 Key 脱敏显示列；操作列增加"启用/禁用"+"删除"；禁用行背景变红 |
| API Key 管理 - 删除 | 二次确认弹窗，提示不可撤销 |
| Layout Header | 显示头像+用户名+角色标签（替代"角色: admin"） |
| 全局 | Toast 通知（操作成功/失败反馈）；按钮 loading 态；必填字段 `*` 标注 |

### 后端

| 端点 | 说明 |
|------|------|
| `PUT /api/v1/api-keys/:id` | 启用/禁用 API Key（body: `{enabled: bool}`） |
| `DELETE /api/v1/api-keys/:id` | 删除 API Key（不可撤销） |
| `PUT /api/v1/users/:id/password` | 重置用户密码（body: `{password: string}`） |

## 3. 详细设计

### 3.1 程序管理 - 创建弹窗

```
┌─────────────────────────────────────────┐
│ 注册新程序                               │
│                                         │
│ ┌─ 基本信息 ──────────────────────────┐ │
│ │ 项目名 *    程序名 *                │ │
│ │ [hermes]    [deploy]                │ │
│ │                                     │ │
│ │ 二进制路径 *                         │ │
│ │ [/usr/local/bin/deploy]             │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─ 审批配置 ──────────────────────────┐ │
│ │ 审批人 *                             │ │
│ │ [👤 张三 (user) ▼]  ← 下拉选用户    │ │
│ │ 此程序的执行工单将自动分配给该审批人   │ │
│ │                                     │ │
│ │ 超时时间                             │ │
│ │ [300] 秒                            │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ ┌─ 参数白名单（可选）──────────────────┐ │
│ │ 定义 AI 提交工单时可传的参数名称，    │ │
│ │ 不在白名单中的参数将被拒绝            │ │
│ │                                     │ │
│ │ 参数名        说明           操作    │ │
│ │ [env]    [目标环境]            [✕]   │ │
│ │ [count]  [执行次数]            [✕]   │ │
│ │ [dry_run] [是否试运行]         [✕]   │ │
│ │                                     │ │
│ │ [+ 添加参数]                        │ │
│ └─────────────────────────────────────┘ │
│                                         │
│                      [取消]  [注册中...] │
└─────────────────────────────────────────┘
```

**参数白名单 key-value 表单规则：**
- 每行一个参数：`[参数名]` + `[说明]` + `[删除按钮]`
- 底部 `+ 添加参数` 按钮追加新行
- 提交时前端构造 `{参数名: 说明}` 的 JSON 发送给后端
- 参数名不能重复，前端校验
- 空行自动忽略

**审批人下拉规则：**
- 页面加载时调用 `GET /api/v1/users` 获取用户列表
- 下拉项显示：头像首字 + 用户名 + 角色
- 提交时传 `approver_id`（数字），后端逻辑不变

### 3.2 程序管理 - 编辑弹窗

```
┌──────────────────────────────────┐
│ 编辑程序: hermes/deploy           │
│                                  │
│ ┌─ 启用状态 ───────────────────┐ │
│ │  [●═══] 启用                  │ │
│ │  禁用后 AI 将无法提交此程序的工单│ │
│ └──────────────────────────────┘ │
│                                  │
│ 审批人                           │
│ [👤 张三 (user) ▼]              │
│                                  │
│ 超时（秒）                        │
│ [300]                            │
│                                  │
│               [取消]  [保存]      │
└──────────────────────────────────┘
```

**改动：**
- `enabled` 字段从原生 `<input type="checkbox">` 改为 Switch 组件
- Switch 下方增加说明文字
- 审批人改为下拉选择（同创建弹窗）

### 3.3 程序管理 - 表格

| 列 | 改动 |
|----|------|
| 项目/名称 | 主行显示 `project/name`，副行显示二进制路径（灰色小字） |
| 审批人 | 头像首字圆圈 + 用户名（从裸 ID 改为可读名字） |
| 超时 | 新增列，显示 `300s` |
| 状态 | 不变 |
| 操作 | "编辑" + "启用/禁用"快捷按钮（根据当前状态切换文案） |

**数据获取：** 页面加载时同时请求 `GET /programs` 和 `GET /users`，前端用 `approver_id` 匹配用户名做展示。

### 3.4 用户管理 - 创建弹窗

新增"确认密码"字段。前端校验规则：
- 用户名不能为空
- 密码不能少于 6 位
- 两次密码输入必须一致
- 校验不通过时在字段下方显示红色提示

### 3.5 用户管理 - 表格

- 去掉 ID 列
- 用户名列改为：头像首字圆圈 + 用户名
- 当前登录用户（JWT 中的 user_id 匹配）行背景高亮（淡黄色），操作列显示 `—`
- 其他用户操作列显示"重置密码"链接
- 点击"重置密码"弹出小弹窗输入新密码

### 3.6 API Key 管理

**表格改动：**
- 新增 Key 脱敏列：`lg_a3f8…d2e1`（前 6 位 + … + 后 4 位）
- 操作列根据状态显示：
  - 启用中 → "禁用"
  - 已禁用 → "启用" + "删除"
- 禁用状态行背景变红（`#fef2f2`）
- 页面顶部增加说明文字："用于 AI agent（如 hermes）连接 LoopGuard 的服务账号密钥"

**新增后端端点：**

```
PUT /api/v1/api-keys/:id
  body: { enabled: boolean }
  → 更新 API Key 启用状态

DELETE /api/v1/api-keys/:id
  → 永久删除 API Key
  → 返回: { message: "已删除" }
```

**删除确认弹窗：**
- 标题："确认删除 API Key？"
- 内容：Key 名称 + 脱敏值
- 警告："此操作不可撤销。使用该 Key 的 AI agent 将立即失去访问权限。"
- 按钮："取消" + "确认删除"（红色）

### 3.7 用户重置密码

**新增后端端点：**

```
PUT /api/v1/users/:id/password
  body: { password: string }
  → 重置用户密码
  → admin only
```

**弹窗：** 输入新密码 + 确认新密码，校验规则同创建用户。

### 3.8 Toast 通知

```
成功示例：
┌─────────────────────────────┐
│ ✓ 程序注册成功                │
│   hermes/deploy 已添加到白名单  │
└─────────────────────────────┘

失败示例：
┌─────────────────────────────┐
│ ✕ 创建失败                    │
│   二进制路径不存在: /xxx      │
└─────────────────────────────┘
```

- 右上角弹出，2 秒后自动消失
- 成功：绿色边框 + 背景
- 失败：红色边框 + 背景，包含具体错误信息
- 所有创建/编辑/删除/启用/禁用操作均触发 Toast

### 3.9 Header

```
当前：角色: admin [退出]
改版：[👤 许江泽] [admin] [退出]
```

- 头像首字圆圈 + 用户名 + 角色标签（紫色小 badge）+ 退出按钮
- 用户名和角色从 JWT token 中解码获取（前端已有 user_id 和 role，需新增 username）

### 3.10 按钮 loading 态

所有表单提交按钮在异步操作期间：
- 文案变为"注册中..."/"保存中..."/"创建中..."等
- 按钮禁用（opacity 降低 + pointer-events: none）
- 可选：增加旋转图标
- 操作完成（成功或失败）后恢复

## 4. 不在范围内

- 分页、搜索、筛选（当前数据量小，后续按需加）
- 用户角色修改（第一期只有两角色，场景少）
- API Key 编辑名称（不可变）
- 程序删除（只做启用/禁用）
